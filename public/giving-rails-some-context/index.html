Giving Rails Some Context

Dec 24th, 2012

There has been quite a bit of talk lately around the idea of model decomposition and contexts. In this post, we’ll create a gem that allows you to decompose models into various contexts, and apply only the functionality you need, when you need it.

It’s becoming clear to more and more Ruby developers that we need a method of breaking large models into smaller, more manageable chunks of code. Rails 4 suggests concerns as one way to deal with the problem. Personally, I like the idea of using contexts to bundle up bits of similar code, for both the modularity and to keep models clean.
